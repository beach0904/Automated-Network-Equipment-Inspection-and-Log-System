# 專案需求分析與技術規格書 (PRD)

## 專案名稱：企業級跨區域網路自動化維運與監測系統
**目標職位：** 新世代網路工程師 (Next Gen Network Engineer)
**版本：** v1.0 (實戰應用版)

---

## 1. 專案背景與動機
在電信級或跨廠區（如：高雄小港、前鎮）的網路管理中，傳統的人工巡檢存在以下痛點：
* **效率低下：** 逐台設備登入檢查耗費大量時間。
* **數據零散：** 巡檢結果分散在各個日誌檔，難以進行長期趨勢分析。
* **即時性差：** 發生故障（如斷線、高負載）時無法第一時間獲得量化告警。

本專案旨在透過 **Python 自動化技術** 結合 **資料庫管理**，打造一套可擴充的網路監測解決方案。

---

## 2. 技術棧 (Technology Stack)
本專案採用模組化設計，確保開發效率與系統穩定性：

| 模組 | 使用技術 | 說明 |
| :--- | :--- | :--- |
| **主程式語言** | **Python 3.x** | 核心邏輯處理、數據分析。 |
| **數據存儲** | **SQLite3** | 輕量化關聯式資料庫，模擬 MySQL 實作環境，負責歷史數據存查。 |
| **自動化庫** | **Netmiko / Paramiko** | 模擬 SSH 連線，實現遠端指令下達與數據抓取。 |
| **數據解析** | **Pandas** | 負責大規模數據的清洗、轉換與統計。 |
| **報表產出** | **Openpyxl** | 自動化生成帶有條件格式化（標色）的專業 Excel 報告。 |
| **時效管理** | **Datetime** | 處理巡檢時間戳，確保歷史紀錄的精準性。 |

---

## 3. 系統架構設計
系統分為四個核心階層：

### A. 數據採集層 (Data Collection)
* **機制：** 透過腳本模擬/實作 SSH 連線。
* **輸出：** 抓取設備 `Show` 指令後的原始文字流 (Raw Text)。

### B. 數據解析與 ETL 層 (Processing)
* **功能：** 將非結構化文字轉化為 Python 字典或 DataFrame。
* **清洗：** 處理缺失值、格式化 IP 位址。

### C. 持久化存儲層 (Storage - SQLite3)
* **Table: `device_inventory`** - 儲存固定資產（IP, 地點, 型號）。
* **Table: `performance_logs`** - 儲存動態指標（CPU, RAM, Status, Timestamp）。

### D. 應用與展現層 (Presentation)
* **功能 1：** 自動生成「全區域維運分析報表」。
* **功能 2：** 自動判定 `CRITICAL` (斷線) 與 `WARNING` (高負載) 狀態並告警。

---

## 4. 功能清單 (Feature List)
1.  **批次處理：** 支持一次讀取多個廠區的數據來源。
2.  **異常預警：** CPU 負載 > 80% 或狀態為 Down 時，系統自動標記。
3.  **歷史溯源：** 透過資料庫存儲，可查詢過去 30 天內任何設備的健康狀態。
4.  **視覺化報表：** 生成帶有顏色導引的 Excel 報告，提升主管審閱效率。

---

## 5. 未來擴展性 (Scalability)
* **API 整合：** 對接 Line Notify 或 Telegram Bot 實現即時推送告警。
* **前端展示：** 結合 Flask 或 Django 建立 Web 版網管儀表板。
* **資料庫升級：** 隨業務量擴展，可無痛遷移至 MySQL 或 PostgreSQL。
