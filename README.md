# 網路自動化維運與大數據監測系統 (v2.0.0)

## 專案簡介
本專案專為網路管理環境設計，結合 **Python 自動化技術** 與 **3NF 關聯式資料庫架構**，針對真實 IT 效能指標進行深度監控。系統核心已捨棄模擬數據，全面對接 **Big_data_dataset (共 10,000 筆真實數據)**，實作從數據清洗、結構化存儲到智慧化異常分析的完整維運管線。

## 🛠 核心功能
* **真實大數據整合**：對接 **Kaggle 真實 IT 效能指標**，涵蓋 CPU、記憶體、Disk IO、網路延遲、進程數、執行緒數、快取缺失、設備溫度、功耗及運行時間等 12 項關鍵維運維度。
* **數據清洗與預處理 (ETL)**：針對 10,000 筆真實數據實作自動化清洗邏輯，利用 Pandas 處理缺失值 (NaN) 與格式異常，確保分析結果的精確度。
* **3NF 關聯式資料存儲**：採用 SQLite3 並實作 **第三正規化 (3NF)** 設計，將設備靜態資料與效能日誌分離，有效管理大規模設備資產。
* **智慧化異常診斷**：實作多因子告警引擎（如：狀態碼異常、設備過熱 > 90°C、高 CPU 負載 > 80% 等），自動判定 `CRITICAL` 與 `WARNING` 等級。
* **自動化視覺報表**：生成具備 **全列標色渲染 (Color-Guided Rendering)** 與 **自動欄寬調整** 功能的 Excel 異常追蹤報告，大幅提升管理者決策效率。

## 📂 專案檔案說明
* **`main.py`**: 核心驅動程式，負責資料庫初始化、ETL 數據導入、以及智慧診斷邏輯執行。
* **`source_data/`**: 原始數據存放區。目前已整合 **Big_data_dataset**，內含 10,000 筆來自 Kaggle 的真實 IT 運行紀錄（支援 .csv 與 .xlsx）。
* **`docs/`**: 包含專案需求分析書 (PRD) 及更新日誌 (Changelog)，記錄從 v1.0.0 到 v2.0.0 的技術演進。
* **`v200_真實數據異常報告_YYYYMMDD.xlsx`**: 系統自動產出之最終分析結果，具備異常高亮顯示。

## 📊 數據規格
* **樣本量**：10,000 筆
* **數據來源**：[Kaggle - IT System Performance & Resource Metrics](https://www.kaggle.com/datasets/freshersstaff/it-system-performance-and-resource-metrics)

## 🚀 快速開始
1. 將 Kaggle 下載的數據放入 `source_data` 資料夾。
2. 執行 `python main.py`。
3. 系統將自動初始化資料庫、匯入數據並產出異常分析 Excel 報告。
